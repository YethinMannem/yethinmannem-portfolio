Completed: December 2023
Duration: 3 months
Team: Solo Project

Project Overview

The MIPS Processor Simulator is a comprehensive implementation of the MIPS (Microprocessor without Interlocked Pipeline Stages) architecture in C++. This project demonstrates deep understanding of computer architecture principles by simulating a complete CPU with register management, memory operations, and instruction execution pipeline.

Educational Value

This simulator serves as an educational tool for understanding computer architecture fundamentals, providing hands-on experience with assembly programming, instruction execution cycles, and the inner workings of a modern processor. It bridges the gap between theoretical knowledge and practical implementation of CPU architecture.

Key Features

  • Complete MIPS instruction set implementation (R-type, I-type, J-type)
  • 32 general-purpose registers with proper register file management
  • Memory management with load/store operations
  • Instruction pipelining simulation (IF, ID, EX, MEM, WB stages)
  • Branch prediction and hazard detection
  • Assembly code parsing and execution
  • Real-time register and memory state visualization
  • Performance metrics and cycle counting
  • Error handling and debugging capabilities
  • Comprehensive test suite with sample programs

Technical Implementation

The simulator is built using object-oriented C++ design principles, implementing the MIPS architecture specification with accurate timing and behavior. The codebase features modular design with separate classes for registers, memory, ALU, and control units, making it extensible and maintainable.

Technical Challenges

The main challenges included accurately implementing the MIPS instruction set architecture, managing data hazards in pipelined execution, and ensuring proper memory alignment and addressing. I solved these by thoroughly studying the MIPS specification, implementing robust error checking, and creating comprehensive test cases to validate instruction behavior.

JYP Compiler

A custom programming language compiler built with Lex and Yacc that translates JYP source code into assembly language, demonstrating advanced compiler design principles

Completed: December 2023
Duration: 3 months
Team: Academic Project
View Code Documentation

Project Overview

The JYP Compiler is a comprehensive implementation of a custom programming language compiler that demonstrates deep understanding of compiler design principles. Built using Lex and Yacc (Flex and Bison), this project showcases the complete compilation process from source code parsing to assembly code generation, providing hands-on experience with language design and compiler construction.

Educational Value

This compiler project serves as an educational tool for understanding the fundamental concepts of programming language design and compiler construction. It bridges the gap between theoretical knowledge of compiler design and practical implementation, providing insights into lexical analysis, syntax parsing, semantic analysis, and code generation phases of compilation.

Key Features

  • Custom JYP programming language with C-like syntax
  • Lexical analyzer built with Flex (Lex) for token recognition
  • Parser generator using Bison (Yacc) for syntax analysis
  • Semantic analysis with type checking and scope management
  • Intermediate code generation and optimization
  • Assembly code generation for target architecture
  • Error handling and recovery mechanisms
  • Symbol table management for variable and function tracking
  • Support for basic data types and control structures
  • Memory management and register allocation

Technical Implementation

The compiler is implemented in C++ using Flex and Bison for lexical and syntax analysis respectively. The system follows the traditional compiler architecture with distinct phases: lexical analysis, syntax analysis, semantic analysis, intermediate code generation, optimization, and target code generation. The compiler generates assembly code that can be assembled and linked to create executable programs.

Technical Challenges

The main challenges included designing a coherent language grammar, implementing efficient parsing algorithms, managing symbol tables and scopes, and generating correct assembly code. I solved these by carefully designing the language specification, implementing robust error handling, creating efficient data structures for symbol management, and thoroughly testing the compilation process with various input programs.

Completed: February 2024

Project Overview

The WCAG Compliance Checker is a comprehensive VS Code extension designed to help developers create more accessible web applications by providing real-time feedback on Web Content Accessibility Guidelines (WCAG) 2.1 compliance. This tool bridges the gap between development and accessibility requirements, making it easier for developers to build inclusive applications.

Business Problem

Web accessibility is often overlooked during development due to lack of awareness and tools. Many developers struggle to understand and implement WCAG guidelines, leading to inaccessible applications that exclude users with disabilities. This extension addresses this gap by providing immediate, contextual feedback during the development process, helping teams build more inclusive applications from the start.

Key Features

  • Real-time WCAG 2.1 compliance checking in HTML, CSS, and JavaScript files
  • Contextual error highlighting with detailed explanations and severity levels
  • Automated suggestions for accessibility improvements
  • Support for all WCAG 2.1 success criteria (A, AA, AAA levels)
  • Customizable rule configurations and severity thresholds
  • Integration with VS Code's Problems panel and status bar
  • Quick-fix suggestions for common accessibility issues
  • Documentation links to official WCAG guidelines
  • Performance optimized for large codebases
  • Multi-language support for international accessibility standards

Technical Implementation

The extension is built using TypeScript and leverages VS Code's Language Server Protocol (LSP) for efficient code analysis. It implements a custom parser that analyzes HTML structure, CSS styling, and JavaScript interactions to identify potential accessibility violations. The system uses a rule-based engine that applies WCAG 2.1 success criteria to detect issues like missing alt text, insufficient color contrast, keyboard navigation problems, and semantic HTML violations.

Technical Challenges

The main challenges included creating accurate accessibility rule implementations, optimizing performance for large files, and providing meaningful suggestions that developers can easily implement. I solved these by implementing efficient parsing algorithms, creating a comprehensive rule engine based on WCAG specifications, and developing a suggestion system that provides actionable, contextual recommendations with code examples.
